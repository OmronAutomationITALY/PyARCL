AMRSimTools module

AMRSimTools.program(main, repeatTime=0, arguments=())[source]

Function that is used to ease the multithreading process.

Function that allows easy handling of multithreading requests for multiple functions. This could be used to easily send missions in a parallel way or multithreads that perform checks to the Robots state and / or store queue information.

Parameters

  • main (function()) – Function preferably with no arguments that will be used to start the multithreading process. If a repeat time is not specified, this function will need to contain the main event loop or the sequence of actions will end after the function completition.

  • repeatTime (int) – If this positive value is specified, then the main function will be instanced over and over with a time interval equal to the number of seconds specified. This can be used to circumvent the event loop in case where the takt time is not fixed but the events still need to start occurring at a specified pace.

  • arguments (tuple) – Tuple containing the arguments of the main function in case it wasn’t a no-argument function. To use with care.
